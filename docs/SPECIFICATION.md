# SPECIFICATION

## コンセプト

自作したライブラリをスニペットに登録し、エディタから簡単に呼び出せるようにしたい。

## コンセプトを実現するために

1. 自作したライブラリのコードがどこに該当するかを判断できるようにする
2. ライブラリの情報(何の言語か、どこに配置されているか)を設定値として管理する
3. 環境に依存せずに使用できるようにする

## 実装内容

### 1. 自作したライブラリのコードがどこに該当するかを判断できるようにする

- ライブラリフォルダ内の指定した拡張子のソースを対象にライブラリコードを抽出する
- ライブラリとなるコードには、あらかじめ指定した開始行と修了行のランドマークをつけておき、その間のコードを抽出する
- ライブラリとなるコードには、スニペットに登録する際のキーやプレフィックスを記載しておき、その情報を使用してスニペットjsonに記載する
  - スニペットjsonに記載する際、どのsetting.ymlを参照したかをわかるようにする
  - スニペットのキーはユニークとなるようにし、競合しないようにする

> コンフィグ、設定に関しては、[{対象のプロジェクト}/.library_setting_registration/setting.yml] に記載して管理する。

### 2. ライブラリの情報(何の言語か、どこに配置されているか等)を設定値として管理する

- 対象となるライブラリフォルダを参照し、ライブラリコードを探索する
- ライブラリの言語設定を参照し、スニペット記載時はその言語に対応する{言語名}.jsonに記載する

> ライブラリの情報に関しては、[{対象のプロジェクト}/.library_setting_registration/setting.yml] に記載して管理する。

### 3. 環境に依存せずに使用できるようにする

- 実行するデバイスごとに設定ファイルを残せるようにする
  - PCごとにvscodeやcursorのインストール先が異なるので、デバイスごとに各エディタのスニペット配置フォルダを記載できるようにする
  - vscodeをフォークしたエディタであれば設定できるはずなので、対応エディタを容易に追加できるようにする

> デバイスやエディタごとのスニペットパスの設定に関しては、[{対象のプロジェクト}/.library_setting_registration/setting.yml] に記載して管理する。
